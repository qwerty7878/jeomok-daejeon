# P0. 공통 셸

## 목적
모든 페이지가 공유하는 껍데기. 세션, 닉네임, Realtime 연결, 전역 에러.

## 구성

| 요소 | 설명 |
|---|---|
| 헤더 | 로고 `그시절` / 현재 닉네임(수정 가능) / 접속자 수 |
| 푸터 | `이미지 출처` (→ /credits) / 문의 |
| 토스트 | 성공·에러 알림 (우상단, 3초) |
| 전역 모달 | 닉네임 설정, 확인 다이얼로그 |

## 세션 (SES)

| ID | 기능 | 상세 |
|---|---|---|
| SES-01 | sessionId 발급 | 최초 진입 시 UUID v4 생성 → `localStorage.sessionId` |
| SES-02 | 닉네임 저장 | `localStorage.nickname` |
| SES-03 | 닉네임 모달 | 닉네임 없으면 강제 노출. 2~8자, 금칙어 필터, 랜덤 제안 버튼 |
| SES-04 | **로그인 없음** | 계정·이메일·소셜 전부 없음 |

## Realtime 연결 (RT)

| ID | 기능 | 상세 |
|---|---|---|
| RT-01 | 로비 채널 구독 | `lobby` 채널 — 방 목록 갱신, 로비 채팅, 접속자 Presence |
| RT-02 | 방 채널 구독 | `room:{code}` — 방 입장 시 |
| RT-03 | **재연결 로직** | 끊김 감지 → 지수 백오프 재구독 → **성공 시 `GET /state` 재동기화** |
| RT-04 | 연결 상태 배지 | 🟢 연결 / 🟡 재연결 중 / 🔴 끊김 |
| RT-05 | **200연결 초과 처리** | 연결 실패 시 "접속자가 많습니다. 잠시 후 다시 시도해주세요" |

## 에러 처리 (ERR)

| 상황 | 화면 |
|---|---|
| 존재하지 않는 방 | "사라진 방입니다" + 로비로 |
| 정원 초과 | "방이 꽉 찼습니다" |
| 게임 중 입장 | "게임이 진행 중입니다. 관전으로 입장할까요?" |
| API 500 | 토스트 + 재시도 버튼 |
| Realtime 끊김 | 화면 상단 배너 (조작은 계속 가능, 재연결 시 동기화) |

## DoD

- [ ] 개발자도구에서 네트워크 끊었다 켜면 자동 재연결 + 상태 복원

# P2. 방 만들기 모달

## 폼

| 항목 | 기본값 | 제약 |
|---|---|---|
| 방 이름 | `{닉네임}의 방` | 1~20자, 금칙어 필터 |

| **방 타입** | 🌐 공개 | 공개 / 🔒 잠금 / 🕶️ 비밀 (3택 1) |
| 비밀번호 | — | **잠금·비밀 선택 시만 노출.** 숫자 4자리 |
| 목숨 | 3 | 2~5 |
| 작성 시간 | 45초 | 30 / 45 / 60 |

## 방 타입 UI (선택지에 설명을 박아넣습니다)

```
🌐 공개방    로비에 보이고, 누구나 들어옵니다
🔒 잠금방    로비에 보이지만, 비밀번호를 알아야 들어옵니다
🕶️ 비밀방    로비에 안 보입니다. 링크로만 들어옵니다
```

## 기능

| ID | 기능 | 상세 |
|---|---|---|
| CR-01 | 생성 | `POST /api/rooms` → `{ code, inviteToken }` → `/room/{code}` 이동 |
| CR-02 | 코드 생성 | 6자리, `0 O 1 I` 제외 (혼동 방지) |
| CR-03 | 비번 해시 | 서버에서 bcrypt. **평문 저장 금지** |
| CR-04 | inviteToken | 방 생성 시 발급, 방 수명과 동일 |
| CR-05 | Rate Limit | IP당 분당 5개 |

## DoD

- [ ] 생성 직후 로비 다른 유저 화면에 실시간으로 뜬다 (비밀방은 안 뜬다)

# P4. 초대 링크 진입 `/join/[code]?t={token}`

## 목적
**카톡으로 날아온 링크를 탄 사람의 첫 화면.** 여기서 마찰이 생기면 그 유저는 영원히 안 옵니다.

## 흐름

```
링크 클릭
   ↓
방 정보 조회 (GET /api/rooms/[code])
   ↓
┌─ 방 없음 ──────────────► "사라진 방입니다" + 로비로
├─ 꽉 참 ────────────────► "방이 꽉 찼습니다" + 로비로
├─ 게임 중 ──────────────► "관전으로 입장할까요?" [관전] [로비로]
└─ 정상
      ↓
   토큰 유효?
      ├─ YES ──► 비번 스킵 ──┐
      └─ NO  ──► 비번 모달 ──┤
                            ↓
                     닉네임 입력/확인
                            ↓
                       /room/{code}
```

## 기능

| ID | 기능 | 상세 |
|---|---|---|
| JN-01 | 방 미리보기 | 방 이름, 현재 인원, 방장 닉네임 표시 → **"내가 아는 사람이 있구나" 확신** |
| JN-02 | **토큰 비번 스킵** | `?t=` 유효 → 비번 검증 생략 ★ |
| JN-03 | 토큰 만료/위조 | 비번 모달로 폴백 |
| JN-04 | 신규 유저 안내 | 처음 온 사람에게 룰 3줄 + 예시 이미지 1장 |
| JN-05 | OG 메타 | 카톡 미리보기: 방 이름 + 대표 이미지 + "제목학원 하러 오세요" |

> **JN-05를 대충 하면 카톡에 링크만 덩그러니 뜹니다.** 클릭률이 반토막 납니다.

## DoD
- [ ] 카톡에 링크 붙여넣으면 썸네일 + 제목 + 설명이 뜬다
- [ ] 링크 탄 사람은 비번을 묻지 않는다
- [ ] 클릭 → 대기실까지 **3탭 이내**

| WR-02 | 카운트다운 | `deadline`(서버 발행)에서 로컬 계산. **클라 시계 신뢰 안 함** |
| WR-03 | 제목 입력 | 최대 40자, 실시간 글자수 표시 |
| WR-04 | 제출 | `POST /submit`. **서버가 `deadline` 검증** |
| WR-05 | 제출 후 수정 | 마감 전까지 가능 (덮어쓰기, `UNIQUE(room,round,player)`) |
| WR-06 | 제출 현황 | `SUBMIT_PROGRESS` → `4/6`. **누가 냈는지 ✓만 표시, 내용은 절대 안 보냄** |
| WR-07 | **조기 전이** | 전원 제출 → 타이머 무시하고 즉시 VOTING |
| WR-08 | **미제출 처리** | 빈 제목 → 투표 대상 제외 + **자동 최하위 확정** |
| WR-09 | 채팅 제한 | **제출 완료자만 발언 가능** (미제출자는 입력창 잠금) |
| WR-10 | 이미지 로딩 실패 | 대체 이미지 + 해당 라운드 시간 +10초 |
| WR-11 | Tick 트리거 | `deadline` 도달 → `POST /tick` (§Lazy Transition) |

## 치팅 방어 ★

- 제출 내용은 **다른 클라이언트에게 절대 전송되지 않습니다.** `SUBMIT_PROGRESS`는 숫자만.
- DevTools로 네트워크를 까도 남의 제목이 없습니다.
- 마감 후 제출 → 서버가 `deadline <= now()` 로 **거부**.

- [ ] 미제출자는 다음 화면에서 자동 최하위가 된다
